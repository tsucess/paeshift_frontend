================================================================================

PROBLEM FIXED:
- "Maximum update depth exceeded" error in Notificationmodal component
- Error appeared repeatedly in console when displaying notifications
- Caused by React anti-patterns and unstable dependency chains

ROOT CAUSES:
1. Derived State Anti-Pattern
   - Using both useState and useMemo for the same data
   - useEffect updating state that was a dependency of useMemo
   - Created circular dependency and infinite re-renders

2. Unstable Dependencies
   - applyFilter callback with empty dependency array
   - Used in useMemo which depended on it
   - Caused dependency chain to be unstable

3. Performance Issues
   - Reading from localStorage on every render
   - Multiple state sources for same data
   - Unnecessary computations

SOLUTION IMPLEMENTED:
================================================================================

1. SINGLE SOURCE OF TRUTH
   BEFORE: const [filteredNotifications, setFilteredNotifications] = useState([]);
           const filteredNotificationsMemo = useMemo(() => {...}, [...]);
           useEffect(() => { setFilteredNotifications(filteredNotificationsMemo); }, [...]);
   
   AFTER:  const filteredNotifications = useMemo(() => {
               // Calculate directly, no state needed
               return notificationsList.filter(...);
           }, [notificationsList, activeFilter]);

2. CACHED USER ID
   BEFORE: const currentUserId = localStorage.getItem("user_id");
   AFTER:  const currentUserId = useRef(localStorage.getItem("user_id")).current;

3. STABLE CALLBACKS
   BEFORE: const applyFilter = useCallback((filter, list) => {...}, []);
   AFTER:  Inline logic directly in useMemo, no callback needed

4. MEMOIZED EVENT HANDLERS
   const handleFilterChange = useCallback((filter) => {
       setActiveFilter(filter);
   }, []);

   const handleReadNotification = useCallback((notificationId) => {
       // Handler logic with proper dependencies
   }, [currentUserId, markingAsRead, refetchNotifications, refetchNotificationsLocal]);

5. MEMOIZED UTILITIES
   const groupedNotifications = useMemo(() => {
       // Group notifications by date
   }, [filteredNotifications]);

   const getDateLabel = useCallback((date) => {
       // Format date label
   }, []);

KEY LEARNING:
================================================================================
The "Maximum update depth exceeded" error is a common React anti-pattern.
It occurs when:
- setState is called in useEffect that depends on the state being updated
- Callbacks with empty dependencies are used in other hooks
- Multiple state sources exist for the same data

The solution is to:
- Use useMemo for derived state (not useState + useEffect)
- Keep dependency arrays minimal and stable
- Cache expensive computations
- Use useCallback for event handlers
- Use useRef for values that don't trigger renders
